Another milestone in podcasting. Here is an update on the new features.

jPodder 0.8

This version of jPodder is improving the  usability with instant status feedback, Podcast life-cycle management,  download improvements and easy migration from other iPodder clients. A detailed description of the features is provided below.

Overview status bar

In personal feeds, The overview status bar provides instant feedback on how many feeds are subscribed to, the total number of marked enclosures, the next scheduled download time, the number of files in a feed (When selected), the total number of bytes consumed by a feed, the currently selected player.

Action bar

Located at the top of the personal feeds tab, the action bar reports the most relevant actions. This gives you instead feedback on what is currently happening and which task is reporting a finished job. (Many tasks operate simultaneously in jPodder).

Podcast Life-Cycle & "Ghost feeds"

The enclosure/preview panel, is now also a file panel, you can now see both the enclosures and the previously downloaded files for the feed. If a file is both an enclosure and visible on the folder, it is displayed as an enclosure. (Highlighting is blue). If a file is not an enclosure it's highlighted in orange. You can choose to remove the files (Both from disc and player) or add them to the player if missing. The marking column is now context sensitive, for files on disc, marking allows deletion of the files. (Files on the disc are auto-marked). Files can also be deleted individually or by selecting multiple rows. How? Files in the folder are displayed automatically. 

  • To deleted marked files select a feed, Mark the enclosures and files, right-click on the feed-table-> delete marked. 
  • To delete a single or multiple file (regardless of marking) right-click on the file-table -> delete selected...
  • To add a missing file to the player right-click on the feed-table -> update player... 

jPodder will also scan the root folder for folders corresponding to feeds, which are not in the personal feed list. The "ghost feeds" can be deleted or added to the player. As the URL of the feed is "unknow" it can  be "looked-up" in the directory. (The ghost feed folder name, is used as a query in the directory).

Single download & downloads resume

Download has been extended with support for single-file download.  This provides easier access to the downloading, not needing to initiate a subscribed feed download. (Pressing Download Now!) How? When right-clicking on an enclosure, a popup menu will show and  you can select to download the enclosure. 

Partially downloaded files are automatically resumed. The continuation of the download is transparent, no special action is required from the user.

Direct Play

You can play a file directly in the selected player. Simply right click on the enclosure and select "play". This is only supported for already downloaded files. (On disc column is flagged). 

URL Decoding

Feed URL's are now properly decoded using the URL decoding standard RFC This gets rid of odd looking encoding strings in titles and file names.

Cache learning

You can now instruct jPodder to update the cache with files which are in the folder but might be missing from the cache. (In case the cache.xml file got lost or damaged). You can tell jPodder which cache file to use. How? Settings->Download, check mark the option to "update cache entries".

Import

jPodder can import the settings and feeds file from other iPodder-Lemon in case you want to start using  jPodder and have used iPodder-lemon before. How? In Settings->Import , Just point to the iPodder folder and click import, you will see an overview first.

BitTorrent

Share ratio and total downloaded bytes is now visible in the download panel.

MP3

The latest version of the MP3 library (vd Heide) is now included resulting in better MP3 support.

jPodder 0.7 release:

Welcome to the 0.7 release of jPodder. This release optimizes some existing features, aswell as introduces new advanced stuff. 

Optimized enclosure previewing:

The RSS feed is not parsed automatically when selecting a feed in the feed table. The user has to press the preview/refresh button to parse the RSS file and view the enclosures inside. The file is only parsed if the last-modified date as returned by the server is not changed. The enclosures inside the RSS feed are only queried if jPodder hasn't done some previously. The state of the RSS feed and enclosures are "remembered" by jPodder even when restarting the application.

These changes have major impact on response times and the overall number of network queries like HTTP HEAD.  It benefits the user to a great extend. 

GUI Enhancements:

Various parts of the GUI have been enhanced: 

- Directory: 1) New icons, 2) Trimming of feed title in add button. 3) Subscribed feed has need icon

- Personal Feeds: 1) It's now possible to move feeds up and down the list with new buttons to the left of the feed table.  

- Settings : 1) The settings panel is now a tree structure, which looks nicer and prepared for expansion when adding new settings.

- Download : 1) Download time is displayed. 2) Corrected some wrong displaying of the progress bar. 

Windows Media Player Support:

jPodder can now stored downloaded media files into windows Media Player. The preferred player can be selected in the settings panel -> media. (Limitation)Please note the implementation can not detect if a file already exists in WMP. 

bitTorrent support

Besides multiple concurrent downloads, jPodder now supports bitTorrent downloads. After jPodder has downloaded a file ending on .torrent, it will automatically be added to the bitTorrent download table. This table is the lower pane in the download panel. 

The bitTorrent will automatically seed after the download completed. Please note the bitTorrent Azureus module create a subdirectory in the users home directory to store configuration information. 

ID3v2 MP3 Viewing: 

Basic support of ID3v2 viewing. jPodder can display ID3 information as supplied in the MP3 file.

iPodder 0.2 release:

I stepped the release number by a decimal, I believe it's worth it with the following new features/improvements: 

System tray

iPodder can now be minimized in the system tray. iPodder can be brought forward by single-clicking the system tray icon. 

GUI

The gui now contains Tab panels for Feeds, Log and Settings. The GUI look & feel is based on the JGoodies plastic L&F.

Feed Selection

  1. Feeds can be selected for download or skipped. 
  2. If selected, the number of downloads can be set. (The default is 1).
  3. The feeds settings are now stored and retrieved in XML (feeds.xml).

Scheduling

The scheduler is improved in many aspects. 

  1. You can turn it on or off. This can be done in the system tray or in the settings panel.
  2. When stopping, this is a hard stop (It kills the scanning  process), this can result in partially saved files and unclosed objects. It is recommended, to limit the use of this feature as the effects are unknown. (Sun has deprecated this function in the Thread API). 

Feed scanning

The scanning of feeds has been rebuild from scratch. iPodder now performs tasks in the following order (This can be witnessed in the logging pane).

  1. Collect enclosures from a feed.
  2. Extract a folder name from the first feed title.
  3. Mark the "known" enclosures from either the cache register or the download folders.
  4. Download the unmarked enclosures (Limited by a maximum).
  5. Store the new downloads in the cache file
  6. Store the newly downloaded enclosure in the player (iTunes is only supported for now).

Settings

Settings have been greatly improved with support for: 

  1. Download folder selection 
  2. HTTP Proxy Settings (Needs more settings.
  3. Scheduler settings

Settings are stored in the feeds.xml file. 

Caching

Caching is now also XML based, all resolved enclosures are cached for further querying.

Installation:

This is a JAVA application, which supports the win32 version of iTunes. It will not work on any other platform (yet). To run the application simply make sure that the Java Runtime Environment is installed. It's easy to check when typing:

java -version

When installed, the command above will reply with the java version on your system, otherwise install java from:

http://java.sun.com

To run the application, follow these steps: 

1) Unzip the file to a directory of choosing. [dir]

2) goto the [dir]\bin directory and type install.cmd, follow instruction and install iPodder on your system. 

2) Open the [dir]\bin feeds.txt file using notepad or other text editor. Change the first line to your directory of choosing. 

3) run iPodder from the start-menu or goto the [dir]\bin directory and type iPodder.cmd 

4) This will start a small window, and access the feeds in your feeds.txt file
